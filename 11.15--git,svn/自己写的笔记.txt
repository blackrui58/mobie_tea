<1>在本地生成一个秘钥
命令行为[ ssh-keygen -t rsa -C  '登录的邮箱地址'] 一路回车下去,

    	id_rsa     私有秘钥
    	id_rsa.pub 公有秘钥

    	打开这个id_rsa.pub文件，注意不要用编辑器打开，要用记事本打开！

<2>把秘钥放在github上(绑定)(也就是将本地的git同github进行绑定)
在设置中,找到放置秘钥的位置,这里用的ssh秘钥
在配置中,

title 随意 英文
key:要把所得的秘钥(共有秘钥)复制粘贴上去！


<1>对本地git进行设置,设置邮箱和用户名
命令行为
[设置邮箱] [git config --global user.email "自己要用的github的邮箱名,如{278}"]
[设置用户名] [git config --global user.name "自己的要用的github的账户名::如自己的{blackrui58}用户名"]
//若要更换新的github的话,则需用新的github的邮箱和用户名//那么用上面的命令再次对本地的git进行设置,会产生覆盖的
//用 [ git config -l ]命令可以查看我们对本地git的配置


//git 的配置邮箱
//git 生成的秘钥邮箱
//github的邮箱
//三者必须保持一致

//若要更换github账号,将本地代码提交到新的github上面去,需要以下几个操作
//需要全部替换
//1.删除本机的秘钥,重新生成一个秘钥,并将共有秘钥绑定到github上面
//2.重新对git进行配置
//3.登录新的github账号


电脑的秘钥只能生成一次,再次运行生成秘钥的命令,不会产生覆盖,仍会保留原来的秘钥
所以要想生成新的秘钥,先删除旧的秘钥,然后再生成新的秘钥
